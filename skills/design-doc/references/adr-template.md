# ADR テンプレート

新しい ADR を作成するときはこの構成に従う。

```markdown
# ADR-NNN: [タイトル]

## ステータス

承認済み — [日付]

## コンテキスト

[この決定が必要になった背景。何が問題で、何を解決したいのか。]

## 検討した選択肢

### 選択肢1: [名前]
- **メリット**: ...
- **デメリット**: ...

### 選択肢2: [名前]
- **メリット**: ...
- **デメリット**: ...

## 決定

[何を選んだか、一言で。]

## 理由

[なぜその選択肢を選んだのか。トレードオフをどう評価したのか。]

## 影響

[この決定によって何が変わるか。今後気をつけるべきことがあれば。]
```

## 記述のポイント

- **コンテキスト**は「なぜこの決定が必要だったか」が伝わるように書く。後から読む人は会話の経緯を知らない
- **検討した選択肢**は実際に議論したものだけ。網羅性より正直さを重視する
- **理由**が最も重要なセクション。「チームの経験」「プロジェクトの規模」「将来の拡張性」など、判断の根拠を具体的に書く
